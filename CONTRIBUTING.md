# Guía de Contribución

Gracias por querer contribuir a este proyecto. Para mantener el orden, seguimos estas reglas:

## Flujo de Trabajo (Git Flow)
Utilizamos un flujo simplificado:
- `main`: Rama de producción (protegida).
- `feature/nombre-de-la-caracteristica`: Ramas para nuevas funciones.
- `fix/nombre-del-error`: Ramas para correcciones.

## Conventional Commits
Todos los commits deben seguir este formato:
`<tipo>: <descripción corta>`

Tipos permitidos:
- `feat`: Nueva característica.
- `fix`: Corrección de errores.
- `chore`: Tareas de configuración, dependencias, etc.
- `docs`: Cambios en documentación.

## Pull Requests
Toda fusión a `main` requiere obligatoriamente un Pull Request y al menos una revisión por pares (Peer Review).
